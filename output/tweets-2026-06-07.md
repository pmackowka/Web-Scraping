ANTHROPIC WŁAŚNIE OPUBLIKOWAŁ 36-STRONICOWY PORADNIK BEZPIECZEŃSTWA

Jeśli używasz agentów z Claude Code, serwerów MCP lub narzędzi automatyzacji — zwróć uwagę.

Oś czasu ataku się skurczyła. Modele AI kompresują lukę między podatnością a działającym eksploitem z miesięcy do godzin, za kilka dolarów.

Agenty wprowadzają nowe ryzyka: zatruwanie narzędzi, manipulację pamięcią kontekstu.

Najbardziej użyteczna koncepcja z poradnika — test bezpieczeństwa Anthropica: "Czy dana kontrola czyni atak niemożliwym, czy tylko uciążliwym?"

Zautomatyzowani atakujący mają nieograniczoną cierpliwość. Przedrą się przez tarcia takie jak rate limiting i 2FA.

Jak Anthropic radzi zabezpieczać agentów:
→ Traktuj statyczne klucze API jako skompromitowane. Używaj krótkożyciowych tokenów (ważność: minuty).
→ Stosuj "Least Agency": ograniczaj, co każde narzędzie może ROBIĆ.
→ Izoluj agentów przetwarzających niezaufane dane (emaile, strony).
→ Skaluj uprawnienia dynamicznie per task, nie na stałe.

> **Co to znaczy:** To najważniejszy dokument dotyczący bezpieczeństwa agentów AI w tym roku. Dla każdego, kto buduje workflow z Claude Code, OpenCode czy MCP, kluczowe jest wdrożenie zasady najmniejszego uprzywilejowania — agent powinien mieć dostęp tylko do tego, czego potrzebuje w danej chwili. Praktycznie oznacza to używanie krótkożyciowych tokenów zamiast kluczy API na stałe zapisanych w plikach .env. W kontekście agentic workflow warto też sandboxować agentów przetwarzających dane z zewnątrz, np. scrapujące strony czy czytające emaile. To podejście zmienia sposób myślenia — bezpieczeństwo to nie lista zakazów, tylko architektura, w której atak jest z definicji niemożliwy.
