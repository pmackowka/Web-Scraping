### @bcherny
**Boris Cherny** | Data: 2026-06-08 | ❤️ Polubienia: 2 467 | 🔁 192 | 👁 386 481

Widzę wiele benchmarków pokazujących, że Opus jest najlepszym modelem do długotrwałej pracy.

Pięć porad, jak uruchamiać Opus autonomicznie przez godziny/dni:

1. Używaj auto mode dla uprawnień — Claude nie będzie pytał o zgodę
2. Używaj dynamicznych workflowów — Claude może orkiestrować setki/tysiące agentów
3. Używaj /goal lub /loop — żeby Claude kontynuował, dopóki nie skończy
4. Używaj Claude Code w chmurze — możesz zamknąć laptopa (najłatwiej przez desktop/mobile app)
5. Upewnij się, że Claude ma sposób na samodzielną weryfikację: rozszerzenie Chrome do przeglądania, symulator iOS/Android przez MCP dla aplikacji mobilnych, sposób na uruchomienie serwera webowego dla backendu

> **Co to znaczy:** Boris Cherny (twórca Claude Code) zdradza swoją konfigurację do długotrwałych sesji — kluczowe jest przejście na tryb w pełni autonomiczny (auto permissions + dynamiczne workflowy). Dla programisty najważniejsza jest piąta zasada: agent musi mieć pętlę samoweryfikacji, żeby nie generować kodu, którego nie jest w stanie przetestować. W praktyce warto skonfigurować Claude Code z MCP do przeglądarki (Claude in Chrome) i symulatora, żeby mógł sam sprawdzać działanie frontendu i mobile. Auto mode dla uprawnień radykalnie zwiększa produktywność, ale wymaga zaufania do agenta.
